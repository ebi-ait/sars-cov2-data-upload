Obtain a secure submission key

To use the data uploader you need to request a secure submission key, by contacting our helpdesk - virus-dataflow@ebi.ac.uk. In your request, please provide an email address, and, if already created, a Webin submission account. If you don’t have a Webin account, you can register for one here or we can register one on your behalf; please let us know if you would like us to do this. A user support bioinformatician will contact you and provide you with a secure submission key within a working day.

Can I share my submission key? The secure submission key uniquely identifies you and serves as authentication to the uploader file - do not share your submission key unless you want to share file access.

I lost my submission key…? If you have lost or forgotten your secure submission key please email our helpdesk virus-dataflow@ebi.ac.uk who will be able to assist.

I asked for a submission key but didn’t receive it? Please email our helpdesk virus-dataflow@ebi.ac.uk who will be able to assist.

 

Login using a secure submission key

When reaching the data uploader web page, you will use your secure submission key, your name, your email address and your WEBIN Submission Account ID to login to the system. In case you don't have a WEBIN Submission Account ID you can create one here.
Enter the required details in the input fields and click the login button.

I entered the wrong key? If your submission key is invalid an error message will be displayed.

My submission key is not working? Please email our helpdesk virus-dataflow@ebi.ac.uk who will be able to assist.

Why my name and email address is required? It helps us in further tracking your submission. Please contact our helpdesk at virus-dataflow@ebi.ac.uk and they will provide more information on this.

Preparing the Metadata File

A spreadsheet will be provided to you by the user support bioinformatician. Please complete all mandatory fields to enable for submission to be carried out successfully. Each row should represent an individual file to be submitted. Accepted values/format have been indicated for specific fields, including a drop-down list of values to choose from.

Study
Please provide descriptive information about a study to group associated data under. This includes a study alias, a short study name, short description of the project and also a longer description of the project.

Sample
Sample mandatory fields must be completed, this follows the sample checklist ERC000033 (ENA virus pathogen reporting standard checklist). A descriptive sample alias, valid taxonomic ID, scientific name, common name, descriptive sample title, sample description, collection date in the relevant format and geographic location should be provided. For all other mandatory fields, if these values are known and can be provided, then should be specified. Alternatively, if this information cannot be provided, a value of ‘not provided’ can be specified.
Furthermore, recommended fields for checklist ERC000033 have been specified in the final columns of the spreadsheet. If known, these values can also be provided for the registration of sample(s).

Run
This section enables you to describe the experimental metadata surrounding the runs. All mandatory fields should be completed, including a descriptive experiment name, sequencing platform, sequencing instrument, library source, library selection and library strategy. This also includes the name of the file (to be uploaded) and also its md5 checksum value. For guidance in this, see documentation here. An optional description of the library and its name can be provided, e.g. a link to a specific protocol that was used to obtain the data.

Select files to upload

To upload files using the data uploader, you can drag and drop them into the central upload area of the page, or you can click on the upload area to open a browsing window which will allow you to manually select files in your filesystem. Accepted data file formats are .fastq.gz, .fastq.bz2, .fq.gz, .fq.bz2, .bam, .cram. You must also have a metadata file supporting your submission. Accepted metadata file formats are .xls, .xlsx, .xlsm, .csv, .tsv, .txt.

Please upload all files related to your submission, data and metadata.

I have a file in an unsupported format? To submit files in an unsupported format please email our helpdesk at virus-dataflow@ebi.ac.uk.

I have selected/dragged a file in the wrong format? When hovering over the file in the upload area, the top right corner of the file will display a little cross. Click on the cross to remove the file from the upload area. Removing a file from the upload area does not delete your file locally.

I have the right file types but still can’t submit? Please check that all the files related to your submission are present in the upload area. The data uploader will require you to enter at least a metadata file and a data file.

Upload and submit files

When all the files related to your submission are in the upload area, please proceed with upload and submission by clicking the ‘Submit’ button. This will send the files to our secure upload area and notify the user support bioinformaticians of a new submission.

How do I know my submission was successful? When you submit your files, our user support bioinformaticians team is notified and will validate your submission. You will hear from the team by email within 2 working days of clicking the “Submit” button, either with questions relating to data validation or with confirmation of successful submission and advice on accession numbers.
